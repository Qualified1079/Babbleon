Babbleon Scrambler

Adversarial test harness. Scramble a small program's identifiers and string literals with a per-run seed; ship the scrambled puzzle to an LLM solver; unscramble the proposed insertion to check it. Python keywords and syntax are preserved so the puzzle is at all readable.

1. Scramble (you → LLM)

2. Unscramble (LLM → you)

How it works / what's measured

Both scrambling and unscrambling are pure functions of (seed, identifier). The mapping is computed on the fly — no server, no storage. An identifier maps to a 4-word lowercase compound from a 2048-word list, deterministically derived from a seeded hash chain.

Preserved (so the puzzle is solvable): Python keywords, operators, numbers, the {{INSERT}} marker, comments are preserved.

Scrambled (so the LLM has no semantic anchor): all other identifiers (function names, variable names, builtins like print and range), and the contents of string literals (word-by-word).

Caveat: this tests the source-level naming layer only. Real Babbleon enforcement also strips binaries, hides syscall metadata, blocks /proc inspection, and rotates the mapping. A scrambled-source puzzle is the weakest form of the test; if the LLM still can't solve it, the full system is much stronger.
